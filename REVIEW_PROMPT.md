# Self-Review Prompt — Nano Banana Studio

## Задача
Открой и внимательно изучи приложение Nano Banana Studio. Проанализируй UI/UX, функциональность и код. Найди реальные проблемы и улучшения.

## Что запустить
```bash
cd /home/user/BananaTool
npm run dev
# Открыть http://localhost:5173
```

## Что проверить

### 1. Скролл и лэйаут
- Прокрути каждую колонку — нет ли прыжков, обрезания контента, двойных скроллбаров
- Проверь поведение при малой высоте экрана (< 700px)
- Проверь на широких экранах (> 1920px)
- Есть ли горизонтальный скролл на мобильном?

### 2. Генерация изображений
- Пройди полный флоу: загрузи ref-фото → Extract DNA → Build Prompt → Generate
- Работает ли sampleCount > 1?
- Что происходит при ошибке API (нет ключа, лимит)?
- Есть ли индикация прогресса?

### 3. UX мелочи
- Все кнопки имеют hover/active состояния?
- Есть ли feedback при копировании текста (DNA JSON, prompt)?
- Drag & drop для изображений работает?
- Очистка слотов — интуитивно?

### 4. История (колонка 3)
- Скролл истории не ломает layout?
- Клик на элемент истории восстанавливает prompt/DNA?
- Кнопка очистки работает?

### 5. Производительность
- Нет ли лагов при большом количестве изображений в истории?
- Memory leaks при смене изображений в слотах?

### 6. Сравнение с конкурентом (NBS из YouTube)
После изучения нашего приложения — посмотри файлы из папки конкурента и составь список:
- Что есть у них, чего нет у нас
- Что сделано лучше у них
- Что у нас лучше

## Формат ответа
Для каждой найденной проблемы:
- **Проблема**: описание
- **Где**: файл + строка
- **Приоритет**: High / Medium / Low
- **Решение**: конкретный код или подход
